2 · Laufzeit & Paketmanager

Node.js v24.14.0

JavaScript-Laufzeitumgebung, Grundlage für Next.js und alle Build-Werkzeuge.

  • Download: https://nodejs.org → „LTS"-Version empfohlen
  • Installer: .msi für Windows (installiert Node + npm automatisch)
  • Prüfen: node --version

npm v11.9.0

Paketmanager, wird mit Node.js mitinstalliert.

  • Prüfen: npm --version
  • Update (optional): npm install -g npm@latest
Empfehlung: Node.js Version Manager nvm-windows (https://github.com/coreybutler/nvm-windows) ermöglicht den einfachen Wechsel zwischen Node-Versionen, falls mehrere Projekte unterschiedliche Versionen benötigen.

Wichtige npm-Befehle im Projekt

npm install          # Abhängigkeiten installieren (nach git clone)
npm run dev          # Entwicklungsserver starten (Port 3002)
npm run build        # Produktions-Build → out/
npm run lint         # ESLint-Prüfung

GitHub CLI (gh) Neu

Das offizielle Kommandozeilen-Tool von GitHub – ergänzt Git um GitHub-spezifische Operationen, die sonst nur über den Browser möglich wären.

  • Installation: winget install --id GitHub.cli
  • Einmalige Anmeldung: gh auth login --web --git-protocol https
  • Prüfen: gh --version

Warum lohnt sich die Installation?

VorteilBeispiel
Repository-Metadaten direkt aus dem Terminal ändern gh repo edit --description "Neue Beschreibung" – kein Browser nötig
Claude Code kann GitHub-Aufgaben vollständig automatisieren Repo-Beschreibungen, Topics, Issues anlegen – ohne Rückfrage und ohne Browser-Umweg; spart Token und Rückfragen
Pull-Requests und Issues aus dem Terminal gh pr create, gh issue list, gh pr merge
Release-Verwaltung gh release create v1.0 --notes "..."
Repository-Informationen abrufen gh repo view, gh run list (CI-Status)
Einmaliger Setup: Nach gh auth login öffnet sich der Browser für die GitHub-Anmeldung – danach ist gh dauerhaft authentifiziert und braucht keine erneute Anmeldung. Der Token wird sicher im Windows Credential Manager gespeichert.

4 · Code-Editor: Visual Studio Code

Primärer Editor für alle Projekt-Dateien (TypeScript, TSX, CSS, Markdown, HTML).

  • Download: https://code.visualstudio.com
  • Version: aktuellste stabile Version empfohlen (Auto-Update aktiv lassen)
  • Installer: VSCodeSetup-x64-*.exe für Windows 64-Bit
  • Option bei Installation: „In PATH aufnehmen" aktivieren → ermöglicht code . im Terminal

Projekt öffnen

Integriertes Terminal

VS Code bringt ein integriertes Terminal mit (Strg + `). Empfehlung: als Standard-Shell Git Bash oder PowerShell konfigurieren.

# Standard-Shell auf Git Bash setzen (in VS Code settings.json):
"terminal.integrated.defaultProfile.windows": "Git Bash"

6 · KI-Assistenz: Claude Code

Claude Code ist der KI-Assistent von Anthropic und wird in diesem Projekt auf zwei Wegen genutzt: als VS-Code-Erweiterung (primär) und als CLI-Tool (optional, für terminal-basierte Workflows).

VS-Code-Erweiterung (primärer Weg)

Die Erweiterung anthropic.claude-code integriert Claude direkt in VS Code. Sie kennt den gesamten Workspace-Kontext, kann Dateien lesen und schreiben, Terminal-Befehle ausführen und Codeänderungen vorschlagen.

  • Installation über VS Code Marketplace oder Befehl (siehe Abschnitt 5)
  • Anmeldung mit Anthropic-Account (claude.ai) erforderlich
  • Öffnen: Seitenleiste → Claude-Icon oder Strg+Shift+P → „Claude"

Claude Code CLI (optional)

Das CLI ermöglicht die Nutzung von Claude Code direkt im Terminal, unabhängig von VS Code.

# Installation (Node.js erforderlich)
npm install -g @anthropic-ai/claude-code

# Starten im Projektverzeichnis
cd C:\Daten\Projects\pbc-erding2
claude

# Verfügbare Slash-Commands im Projekt:
/bbv-ergebnisse     # BBV-Spielergebnisse abrufen und eintragen
/pbced-publish      # Build erstellen und auf GitHub veröffentlichen
/pbced-anpassungen  # Anpassungen.md und .html fortschreiben

Skills (Projekt-spezifische Automatisierungen)

Skills sind wiederverwendbare Anweisungen für Claude Code, die im Verzeichnis .claude/skills/ gespeichert sind. Sie erscheinen automatisch als /skill-name-Befehle.

SkillAufrufZweck

pbced-publish/pbced-publishBuild, Deployment und GitHub-Push
pbced-anpassungen/pbced-anpassungenAnpassungen.md + .html fortschreiben

CLAUDE.md – Projektwissen für Claude

Die Datei CLAUDE.md im Projektverzeichnis enthält dauerhaftes Projektwissen (Design-System, Dateistruktur, Deployment-Befehle). Claude liest sie automatisch beim Start jeder Sitzung.

8 · Deployment-Werkzeuge

Robocopy (Windows-integriert)

Synchronisiert den Build-Output (out/) in das lokale weberding-Projekt:

powershell -Command "robocopy 'C:\Daten\Projects\pbc-erding2\out' 'C:\Daten\Projects\weberding\pbced\pbced2' /MIR /NFL /NDL /NJH /NJS"

Exit-Code 1 = Dateien erfolgreich kopiert (kein Fehler). Exit-Code ≥ 8 = echter Fehler.

FTP-Client (für Produktionsserver)

Für den Upload auf den IONOS-Webserver wird ein FTP-Client benötigt.

  • FileZilla (kostenlos) – https://filezilla-project.org
  • Verbindungsdaten: IONOS-Kundenkonto → Hosting → FTP-Zugangsdaten
  • Zielverzeichnis Entwurf: /pbced/pbced2/
  • Zielverzeichnis Produktion: Wurzelverzeichnis /

9 · Installations-Reihenfolge (Neuaufsetzen)

Bei einem Neuaufsetzen der Entwicklungsumgebung diese Reihenfolge einhalten:

1
Git for Windows installieren
https://git-scm.com/download/win → liefert Git, Git Bash und Credential Manager. Bei Installation „Add to PATH" aktivieren.
2
GitHub CLI installieren
winget install --id GitHub.cli → ermöglicht GitHub-Operationen direkt aus dem Terminal. Anmeldung einmalig via gh auth login --web. Damit kann Claude Code Repo-Beschreibungen, Issues und PRs automatisch verwalten – ohne Browser-Umwege.
3
Node.js (LTS) installieren
https://nodejs.org → LTS-Installer für Windows. Installiert Node.js und npm automatisch. Prüfen: node --version
4
Visual Studio Code installieren
https://code.visualstudio.com → Option „In PATH aufnehmen" aktivieren für code .-Befehl im Terminal.
5
VS-Code-Erweiterungen installieren
Claude Code, Tailwind CSS IntelliSense, ESLint, Prettier via Marketplace oder Sammel-Installationsbefehl aus Abschnitt 5.
6
Repository klonen
git clone https://github.com/chrisgitti/pbc-erding2.git → in gewünschtes Verzeichnis (z. B. C:\Daten\Projects\)
7
Abhängigkeiten installieren
cd pbc-erding2 && npm install → installiert alle Pakete aus package.json (ca. 300 MB in node_modules).
8
Entwicklungsserver starten
npm run dev → öffnet http://localhost:3002 im Browser. Hot-Reload aktiv.
9
FTP-Client einrichten (optional)
FileZilla installieren und IONOS-FTP-Zugangsdaten aus dem Kundenkonto hinterlegen.
